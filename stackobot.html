Project Overview

Stack O Bot challenged me to design and implement two original gameplay mechanics within Unreal Engine. I created a defensive shield system with UI integration and hazardous environmental areas that interacted dynamically with the shield, creating strategic resource management gameplay.

The shield mechanic featured pickups that activated a resource bar in the player UI, with refillable pickups scattered throughout the level. Environmental hazard zones depleted shields before damaging health, encouraging players to manage resources strategically. I also implemented variable movement speed controlled via mouse wheel, allowing dynamic adjustment between precision movement and faster navigation.

As a solo developer, I handled all design decisions, Blueprint implementation, UI integration, and iterative balancing. This project emphasized rapid prototyping, scope management, and creating cohesive mechanical interactions under time constraints.

My Contributions

  • Optimized Blueprint event execution using custom events and event dispatchers to maintain performance during continuous damage calculations
  • Created modular Blueprint architecture allowing easy adjustment of damage values, shield capacity, and movement speed parameters without code restructuring
  • Balanced shield depletion rates, damage tick intervals, and pickup values through data-driven iterative playtesting sessions
  • Created dynamic shield resource UI system using UMG widgets that conditionally appears when shield is activated and updates in real-time
  • Developed refillable shield pickup system with instance-editable variables for resource values and respawn timers
  • Programmed damageable environmental area using trigger boxes with tick-based damage events and dual-damage logic (shield priority, then health overflow)
  • Designed and implemented defensive shield pickup mechanic using Blueprint visual scripting with custom collision detection and player overlap events
  • Implemented variable movement speed system using mouse wheel axis inputs bound to character movement component with clamped speed ranges
  • Integrated shield visual mesh attachment to character skeletal system using socket-based component attachment with proper bone hierarchy navigation

Challenges & Solutions

Challenge 1: Shield Balance

Player Feedback: "My shield drains way too fast—it feels pointless to pick one up" and later, "Shields last forever now, there's no challenge anymore."

The Solution: I implemented instance-editable Blueprint variables for all balance parameters, enabling rapid iteration during playtesting sessions. Through systematic testing of different damage rates, tick intervals, and shield capacities, I discovered the optimal balance: hazard traversal consuming 60-70% of shield capacity. This created meaningful resource cost without guaranteeing failure. I supplemented the numerical balance with comprehensive feedback systems—color-coded UI, audio cues at depletion thresholds, and particle effects—ensuring players always understood their resource state. This experience reinforced that effective balance requires observable player behavior data, not just theoretical number adjustments.

Challenge 2: Shield Visual Attachment

Player Feedback: "The shield looks weird floating around the character and clips through their body when they move."

The Solution: I implemented socket-based attachment to the character's skeletal mesh using Unreal's Skeleton Editor. After identifying the optimal attachment bone (chest/spine), I created a custom socket with precise transform offsets and used "Snap to Target" attachment rules to ensure the shield moved naturally with all character animations. This resulted in visually seamless shield positioning that felt integrated with the character rig.

Challenge 3: Feature Scope Management

Player Feedback: "The game has lots of ideas but nothing feels fully finished" and "Some mechanics seem half-baked or buggy."

The Solution: I conducted an honest assessment of development progress and made the difficult decision to reduce scope from five ambitious mechanics to two fully-realized systems. Focusing exclusively on the shield system and environmental hazards allowed me to deeply iterate on core interactions, resulting in polished, thoroughly tested mechanics that worked cohesively together. This taught me that delivering complete, bug-free systems creates better player experiences than numerous half-finished features—a lesson that has shaped all my subsequent project planning.

What I Learned

This project taught me that delivering two polished, fully-functional mechanics creates better player experiences than multiple half-finished systems. I now prioritize feature completion over quantity and build realistic timelines with buffer for iteration rather than optimistic schedules packed with aspirational features.

Working with Unreal's skeletal mesh system strengthened my ability to research unfamiliar engine systems independently and troubleshoot complex technical issues—skills that transfer directly to professional development. I learned to dive into documentation, experiment with different implementation approaches, and iterate until finding optimal solutions.

Implementing interconnected mechanics showed me that engaging gameplay emerges from systems that interact meaningfully rather than exist in isolation. The relationship between shield pickups, hazardous areas, and resource management created strategic depth and emergent decision-making that standalone features couldn't achieve.

Finally, designing clear feedback systems—color-coded UI, audio cues, and particle effects—reinforced that technical implementation means nothing if players can't intuitively understand system states. Players need immediate comprehension of game mechanics to make informed decisions, making visual communication as critical as the underlying code.
